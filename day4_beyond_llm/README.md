# Day 4: Beyond LLMs — RAG & Agentic AI

Welcome to Day 4! Today we learn how to connect LLMs to databases and build smart, self-acting AI agents.

## Today's Agenda

- **Part 1: Intro to RAG**
  - What is RAG? (Giving AI a search engine for facts)
  - Why LLMs need external knowledge (outdated data, private files)
  - Stopping AI hallucinations (grounding answers in real documents)

- **Part 2: Embeddings & Vector Databases**
  - What are embeddings? (Converting words into numbers/meaning)
  - Semantic Search (Searching by meaning, not just exact keywords)
  - Chunking (Splitting files into bite-sized pieces)
  - FAISS (A fast database built for searching vector meaning)
  - Retrieval Pipeline (Step-by-step path from question to answer)

- **Part 3: Agentic AI Architecture**
  - What is an AI Agent? (Using an LLM as a brain to make decisions)
  - Planning (Breaking big goals into smaller steps)
  - Memory (Saving chat history and user context)
  - Tools (Letting LLMs use calculators, search, and run code)
  - Self-Critique (Checking and correcting its own work)

- **Part 4: Hands-on Projects**
  - PDF Chatbot (Ask questions directly to your documents)
  - Tool-Calling Agent (An AI that runs tools to solve tasks)
  - Automation Workflows (Connecting multiple AI steps)

- **Part 5: Best Practices**
  - When to use RAG vs. simple prompts
  - When agents are overkill (using simple code instead)
  - Cost, speed, and safety tips
